How to modify your site after initial setup

Editing Your Website Content

1

Navigate to Your File

Go to your repository and find the file you want to edit (like index.html).

2

Edit the File

Click the pencil icon (✏️) in the top-right corner of the file view.

3

Make Your Changes

Edit the file content in the web editor.

4

Commit Changes

Scroll down, add a commit message, and click "Commit changes".

5

Wait for rebuild

GitHub will automatically rebuild your site with the changes (takes 1-2 minutes).

Changing Pages Settings

1

Go to Settings → Pages

Navigate to your repository's Settings tab, then select Pages in the sidebar.

2

Change Source Branch

If you want to deploy from a different branch, select it from the dropdown.

3

Add Custom Domain

Enter your domain in the "Custom domain" field and follow the verification steps.

4

Toggle HTTPS

If using a custom domain, you can enable or disable HTTPS enforcement.

5

Save Changes

Always remember to save your changes before leaving the page.

Adding a Custom Domain

If you have your own domain name, you can add it in the Pages settings:

  1. Enter your domain in the "Custom domain" field
  2. Configure DNS records with your domain registrar
  3. Wait for DNS propagation (can take up to 24 hours)
  4. Enable HTTPS for your custom domain

GitHub Pages makes it easy to update your site anytime. Simply edit your files and commit changes.
